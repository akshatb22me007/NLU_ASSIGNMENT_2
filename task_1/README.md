# IITJ Word2Vec Pipeline (Task 1)

This README documents the full execution flow in the required order:

1. `extract_text_from_web.py`
2. `remove_iitj_header.py`
3. `find_pdf.py`
4. `extract_text_from_pdf.py`
5. `data_preprocess.py`
6. `corpus.txt` preparation (generated by preprocessing)
7. `train.py` (save models in `models/`)
8. `eval.py`
9. `visualization.py`

## 1) Project Structure (important folders)

- `data/web/`: raw text scraped from websites
- `data/web_clean/`: cleaned web text + extracted PDF text
- `data/pdf_links.txt`: list of PDF URLs used for extraction
- `corpus.txt`: final preprocessed training corpus
- `models/`: trained Word2Vec models (`.model` files)
- `plots/`: generated visualization images
- `word2vec_results.csv`: model training/evaluation summary table

## 2) Environment Setup

Run from this directory:

```bash
cd task_1
```

Use your Python environment (conda or venv), then install dependencies:

```bash
pip install requests beautifulsoup4 pdfplumber nltk gensim pandas matplotlib scikit-learn wordcloud
```

Note:
- `nltk` tokenizers/stopwords are downloaded automatically in `data_preprocess.py` and `train.py`.
- First run may take extra time due to NLTK downloads.

## 3) Run Pipeline in Required Order

### Step 1: Extract text from web pages

```bash
python extract_text_from_web.py
```

Output:
- Text files are saved in `data/web/` as `doc1.txt`, `doc2.txt`, ...

### Step 2: Remove IITJ headers/navigation boilerplate

```bash
python remove_iitj_header.py --input data/web --output data/web_clean
```

Output:
- Cleaned text files in `data/web_clean/`
- Console prints processed file count and boilerplate size

### Step 3: Find PDF links from IITJ domain

```bash
python find_pdf.py
```

Important path note:
- This script writes to `pdf_links.txt` in the data folder.

### Step 4: Extract text from found PDFs

```bash
python extract_text_from_pdf.py
```

Output:
- Reads PDF URLs from `data/pdf_links.txt`
- Saves extracted PDF text as `.txt` files inside `data/web_clean/`

### Step 5 + 6: Data preprocess and prepare `corpus.txt`

```bash
python data_preprocess.py
```

What this does:
- Deduplicates documents from `data/web_clean/`
- Deduplicates sentences
- Cleans/tokenizes text and removes stopwords
- Writes final corpus to `corpus.txt`

### Step 7: Train Word2Vec models and save in `models/`

```bash
python train.py
```

What this trains:
- Model types: `CBOW` and `SkipGram`
- Embedding sizes: `50, 100, 200`
- Window sizes: `3, 5, 8`
- Negative samples: `5, 10`

Total models saved:
- 36 models in `models/`

Also saved:
- `word2vec_results.csv`

### Step 8: Evaluate trained model (`eval.py`)

```bash
python eval.py
```

Default loaded model:
- `models/SkipGram_dim100_win5_neg10.model`

Outputs:
- Similar word lists for selected words
- Analogy results printed in terminal

### Step 9: Run visualization

```bash
python visualization.py
```

Generated plots in `plots/`:
- `cbow_tsne.png`
- `cbow_pca.png`
- `skipgram_tsne.png`
- `skipgram_pca.png`
- `embedding_dashboard.png`
- `wordcloud_corpus.png`

## 4) One-Command Run (optional)

If you want all steps in one sequence:

```bash
python extract_text_from_web.py && \
python remove_iitj_header.py --input data/web --output data/web_clean && \
python find_pdf.py && \
python extract_text_from_pdf.py && \
python data_preprocess.py && \
python train.py && \
python eval.py && \
python visualization.py
```

## 5) Verification Checklist

After pipeline completes, verify:

- `data/web/` is populated
- `data/web_clean/` has cleaned web + PDF text files
- `corpus.txt` exists and is non-empty
- `models/` contains many `.model` files
- `word2vec_results.csv` exists
- `plots/` contains all six image files

## 6) Troubleshooting

### `LookupError` from NLTK

Run:

```bash
python -c "import nltk; nltk.download('punkt'); nltk.download('stopwords')"
```
### `Word not in vocab` in eval

- This is expected for rare words.
- You can change test words inside `eval.py`.

### Plot generation issues

- Confirm models exist in `models/`.
- `visualization.py` currently loads:
  - `models/CBOW_dim100_win3_neg10.model`
  - `models/SkipGram_dim100_win3_neg10.model`
